
## 1. Property Tax: The "Agricultural Status" Application
This is the most critical annual/periodic filing for land-owning farmers. To keep property taxes low, land must be valued based on its **"capacity to produce"** rather than market value.
* **What:** Application for Agricultural Valuation (NMSA 7-36-20).
* **When:** Initially by the **last day of February**. 
* **Frequency:** Once approved, you generally don't need to re-apply annually unless the **use of the land changes** or the County Assessor requests a "re-certification" (usually every 1–3 years depending on the county).
* **Agency:** Your local County Assessor (e.g., Bernalillo, Dona Ana, etc.).

## 2. Tax Filings: Gross Receipts Tax (GRT)
New Mexico does not have a "sales tax"; it has a **Gross Receipts Tax**. 
* **What:** Form TRD-41413 (Combined Reporting System).
* **The Exemption:** Most "unprocessed agricultural products" (raw veg, honey, etc.) are deductible, but you **must still file** to report the gross income and then claim the deduction.
* **Frequency:** Monthly, Quarterly, or Semi-annually (based on your total revenue). 
* **Agency:** NM Taxation and Revenue Department.

## 3. NMDA Specific Permits (Activity-Based)
If you sell at markets or process food, you have specific "as-needed" or annual filings:
* **Farmers Market Inspection Certificate:** Required annually to sell at many state-sanctioned markets.
* **Scale Certification:** If you sell products by weight (e.g., "per pound"), your scales must be registered and inspected annually by the NMDA **Weights and Measures** department.
* **Homemade Food Act (2021):** Small farmers selling "low-risk" foods (jams, breads) no longer need a permit but must have a **Food Handler Card** (valid for 3 years).

## 4. Federal/USDA Filings (Required for Programs)
Even for "state" farming, federal filings are often required to remain eligible for state-administered disaster relief or grants.
* **Crop Acreage Report (FSA-578):**
    * **What:** Documents exactly what you planted and where.
    * **When:** Deadlines vary by crop (e.g., July 15 for most summer crops, August 15 for beans).
    * **Frequency:** Yearly.
* **Agricultural Census:** Every 5 years (next one is 2027), all farmers are legally required to respond to the USDA NASS survey.

1. The Backend (app.py)

This script handles the web server and the heavy lifting of geospatial conversion.
Python

from flask import Flask, render_template, request, jsonify, send_file
import geopandas as gpd
from shapely.geometry import Polygon
import os
import zipfile

app = Flask(__name__)

@app.route('/')
def index():
    return render_template('index.html')

@app.route('/export', methods=['POST'])
def export_shapefile():
    data = request.json
    coords = data['geometry']['coordinates'][0]
    props = data['properties']
    
    # 1. Create the Geometry
    poly = Polygon(coords)
    
    # 2. Build the GeoDataFrame with USDA-required attributes
    # Note: FSA typically wants specific codes (e.g., '0011' for Corn)
    gdf = gpd.GeoDataFrame([props], geometry=[poly], crs="EPSG:4326")
    
    # 3. Project to the standard USDA coordinate system (NAD83 / Albers)
    gdf = gdf.to_crs("EPSG:5070")
    
    # 4. Save components to a temporary folder
    temp_dir = "temp_export"
    os.makedirs(temp_dir, exist_ok=True)
    base_name = "FSA_Report_2026"
    shp_path = os.path.join(temp_dir, f"{base_name}.shp")
    
    # Export to Shapefile
    gdf.to_file(shp_path)
    
    # 5. Zip it (Shapefiles require .shp, .shx, .dbf, and .prj to be valid)
    zip_path = f"{base_name}.zip"
    with zipfile.ZipFile(zip_path, 'w') as zipf:
        for ext in ['.shp', '.shx', '.dbf', '.prj']:
            zipf.write(os.path.join(temp_dir, f"{base_name}{ext}"), f"{base_name}{ext}")
            
    return send_file(zip_path, as_attachment=True)

if __name__ == '__main__':
    app.run(debug=True)

import requests

@app.route('/get_parcel', methods=['POST'])
def get_parcel():
    data = request.json
    lat, lng = data['lat'], data['lng']

    # Using a sample New Mexico WFS Service (NM RGIS or State Land Office)
    # Note: Replace with specific County WFS URL if working in a single county
    wfs_url = "https://mapserver.nmstatelands.org/arcgis/services/LandStatus/MapServer/WFSServer"
    
    params = {
        "service": "WFS",
        "version": "2.0.0",
        "request": "GetFeature",
        "typename": "LandStatus:Parcels", # Example layer name
        "outputFormat": "application/json",
        "srsname": "EPSG:4326",
        "cql_filter": f"CONTAINS(SHAPE, POINT({lng} {lat}))"
    }

    try:
        response = requests.get(wfs_url, params=params, timeout=5)
        return response.json() # Returns the GeoJSON of the parcel
    except:
        return jsonify({"error": "Parcel not found"}), 404

def get_boundary(lat, lng):

    query = f"""
    [out:json];

    response = requests.get(overpass_url, params={'data': query})

import requests

def get_snapped_geometry(lat, lng):

import os
import zipfile
import geopandas as gpd
from shapely.geometry import shape

def create_fsa_zip(geojson_data):
    # 1. Setup paths
    output_dir = "exports"
    os.makedirs(output_dir, exist_ok=True)
    filename = "FSA_578_Export_2026"
    
    # 2. Process Geometry
    geom = shape(geojson_data['geometry'])
    props = geojson_data['properties']
    
    # 3. Create GeoDataFrame
    # Use EPSG:4269 (NAD83) or EPSG:5070 (Albers) for USDA compatibility
    gdf = gpd.GeoDataFrame([props], geometry=[geom], crs="EPSG:4326")
    gdf = gdf.to_crs("EPSG:4269") 
    
    # 4. Save to Shapefile (writes .shp, .shx, .dbf, .prj)
    shp_base_path = os.path.join(output_dir, filename)
    gdf.to_file(f"{shp_base_path}.shp")
    
    # 5. Package into ZIP
    zip_path = f"{shp_base_path}.zip"
    with zipfile.ZipFile(zip_path, 'w') as fsa_zip:
        for ext in ['.shp', '.shx', '.dbf', '.prj']:
            fsa_zip.write(f"{shp_base_path}{ext}", f"{filename}{ext}")
            
    return zip_path

        'geometryType': 'esriGeometryPoint',

        'outFields': '*', 'returnGeometry': 'true', 'f': 'geojson'
    }

1. Microsoft Azure: The "Easiest for Flask"

2. Handling the SQLite Database on Azure

Since you are using SQLite, there is one critical "gotcha" for cloud hosting: Persistence.

    The Problem: By default, Azure App Service files are "ephemeral." If the server restarts, your SQLite .db file (and your registered users) might vanish.

    The Fix: Go to Configuration > Application Settings in the Azure portal and add:

        WEBSITES_ENABLE_APP_SERVICE_STORAGE = true

    This ensures your nmagtech_2026.db file stays put even if the app reboots.

An FSA-578 is the USDA Farm Service Agency's "Report of Acreage" form,
used to certify planted crops, intended use, and acreage to determine eligibility for federal programs and disaster relief. Farmers must submit it by deadlines (commonly July 15) for farms with a pre-assigned Farm Number, including farm/tract/field numbers, planting dates, and irrigation practices. 
Key Components of an FSA-578 Form:

    Producer Information: Operator name, address, and share percentage.
    Farm/Tract/Field Info: Official farm number, tract number, and field/CLU (Common Land Unit) number.
    Crop Details: Crop name, type, intended use (e.g., grain, fiber), and total acres.
    Status Codes: Indicator for crop status (e.g., Initial, Subsquent).
    Map: A, B, C... subfield mappings if a field has multiple crops.
    Signature: Must be signed by the producer to certify accuracy. 

Example Entries (Item 14/15 on Form):

    Field 1: Corn - Yellow, Grain, Irrigated, 100 acres.
    Field 2a: Soybeans, Common, Non-irrigated, 50 acres.
    Field 2b: Wheat, Winter, Grain, Non-irrigated, 25 acres. 

Tips for Completion:

    Deadlines: Generally, July 15th for spring-planted crops, but vary by region.
    Documentation: Bring detailed, precise planting records or digital records from precision ag equipment.
    Reporting Mistakes: Inaccurate reporting is hard to correct, so review field boundaries and acreages carefully.
    Support: File at your local USDA Service Center, with an approved insurance provider, or check Farmers.gov to view your records
